Aufgabe 1.2 Grundlagen Cloud-Computing

  • Wie ist der Begriff Cloud entstanden? Wieso heisst es Cloud?

    • Stammt aus Netzwerkdiagrammen, wo das Internet als Wolke dargestellt wurde.
    • Steht für virtuelle, abstrakte IT-Infrastruktur.

  • Wie wird der Begriff Cloud definiert, z.B. gemäss NIST?

    Cloud Computing ist ein Modell für den allgegenwärtigen, bequemen und bedarfsgerechten Netzzugang zu einem gemeinsam genutzten Pool konfigurierbarer Computerressourcen (z. B. Netzwerke, Server, Speicher, Anwendungen und Dienste), die schnell und mit minimalem Verwaltungsaufwand oder Interaktion mit dem Dienstanbieter bereitgestellt und freigegeben werden können. Dieses Cloud-Modell besteht aus fünf wesentlichen Merkmalen, drei Dienstmodellen und vier Bereitstellungsmodellen Modellen.


  • Welche Cloud Dienstleistungen kennen Sie?
    • Speicher (Dropbox, Google Drive).
    • Rechenleistung (AWS EC2, Azure VM).
    • Datenbanken (AWS RDS, Google Cloud SQL).
    • KI-Services (Google AI, AWS SageMaker).
    • SaaS (Microsoft 365, Salesforce).

  • Welche Cloud Anbieter kennen Sie?
    • Grosse Anbieter: AWS, Microsoft Azure, Google Cloud.
    • Weitere: IBM Cloud, Oracle Cloud, Alibaba Cloud.

  • Welche Cloud Deployment Modelle kennen Sie?
    • Public Cloud: Wird von Dritten betrieben und für viele Kunden öffentlich bereitgestellt (z. B. AWS, Google Cloud).
    • Private Cloud: Eine Cloud-Umgebung, die exklusiv für eine Organisation betrieben wird, entweder On-Premise oder bei einem Anbieter.
    • Hybrid Cloud: Kombination aus Public und Private Cloud, um Flexibilität und Sicherheit zu bieten. Wir nehmen das beste aus den sachen z.B Private Cloud für kritische Daten und Public Cloud für weniger kritische Daten.
    • Community Cloud: Eine spezielle Cloud für eine bestimmte Branche oder Organisationen mit ähnlichen Anforderungen.

  • Was sind Cloud Service Modelle?
    • Infrastructure as a Service (IaaS): Bereitstellung von Hardware-Ressourcen (Server, Speicher, Netzwerk). Beispiel: AWS EC2, Google Compute Engine.
    • Platform as a Service (PaaS): Entwicklungs- und Laufzeitumgebungen für Anwendungen. Beispiel: Google App Engine, Microsoft Azure App Services.
    • Software as a Service (SaaS): Vollständig gehostete Anwendungen, die direkt genutzt werden können. Beispiel: Gmail, Salesforce, Microsoft 365.
    • On-Premise (On-Prem): Betrieb von Anwendungen und Diensten im eigenen Rechenzentrum.

  • Weshalb soll ich Dienste aus der Cloud beziehen? Was sind die Vorteile?
    Vorteile der Cloud
    • Kosteneffizient (weniger Hardware).
    • Skalierbar (flexible Anpassung).
    • Überall zugänglich (Internetverbindung).
    • Automatische Updates & Sicherheit.

  • Was sind die Nachteile?
    • Abhängigkeit vom Anbieter (Lock-in).
    • Datenschutz & Compliance-Herausforderungen.
    • Internetabhängigkeit (kein Netz, kein Zugriff).
    • Laufende Kosten durch Abo-Modelle.

  • Welche Dienstleistungen werden in Ihrem Betrieb On-Premise (eigenes Rechenzentrum) betrieben?
    • Kritische Geschäftsanwendungen (ERP, Finanzsysteme).
    • Legacy-Systeme.
    • Interne Datenbanken mit hohen Sicherheitsanforderungen.

  • Wie werden technologische Beiträge in der Cloud geteilt bzw. zur Verfügung gestellt?
    • APIs (Schnittstellen für externe Anwendungen).
    • Container & Microservices (Docker, Kubernetes).
    • Repositories & Versionierung (GitHub, GitLab).
    • CDN (Cloudflare, AWS CloudFront für schnelle Bereitstellung).
    • Serverless Computing (AWS Lambda, Azure Functions).

  • Welches sind die 5 Merkmale einer Cloud?
    • On-Demand Self-Service (automatische Bereitstellung).
    • Breiter Netzwerkzugriff (überall erreichbar, es soll schnell für den user sein (Fetter Internet Zugang)).
    • Ressourcen-Pooling (geteilte Infrastruktur, Haufen Server zusammensetzen).
    • Schnelle Elastizität (flexible Skalierung sei es horizontal oder Vertikal).
    • Messbarer Dienst (verbrauchsbasierte Abrechnung, Nutzung wird gemessen und wird geholt was gebraucht werden z.B Datenbank wenn ich sie nur brauche).
